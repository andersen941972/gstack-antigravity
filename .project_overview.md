# Gemini Project Context

## プロジェクトの目的
- `gstack`（AIエンジニアリング・ワークフロー）を Google Antigravity 環境で最適に動作させるためのアダプター・ラッパーの開発。

## 使用している技術
- Node.js (v22.17.1), JavaScript
- gstack (Bunベースの外部リポジトリ)
- Antigravity 独自のワークフロー仕様 (`.agents/workflows/*.md`)

## 主要な設定ファイル・スクリプト
- `setup.js`: プロジェクト全体の初期化（`npm run setup`）
- `src/convert-skills.js`: gstackのスキルをAntigravity形式に一括変換
- `package.json`: ビルドおよびセットアップ・コマンドの定義

## ファイル構造の概要
- `gstack/`: 元のリポジトリ（読み取り専用、素材）
- `src/`: 変換・統合ロジック
- `docs/`: 分析、実装計画、引継ぎ資料
- `.agents/workflows/`: 生成されたAntigravity用ワークフロー群（自律実行対応）

## コーディング規約・制約
- **gstack非汚染原則**: `gstack/` ディレクトリ内のファイルは一切変更せず、外部の `.agents/` 側にのみ成果物を出力する。
- **自律実行優先**: 生成するワークフローには必ず `// turbo-all` を付与し、エージェントの自律性を最大限高める。

## あなたのゴール
オリジナルリポジトリである `gstack` の中身を一切変更することなく（汚さずに）、その内容を読み取って Antigravity専用のワークフロー（`.agents/workflows/*.md`）として出力する仕組みを完成させること。

## 開発プロセス
- ドキュメント駆動: 着手前に必ず `.md` 形式で分析結果と実装計画を更新する。
- ソースコードは `gstack/` を汚染せず、ラップするためのスクリプトを `src/` 配下に構築する。

## コーディング規約
- 変換スクリプトはシンプルに保機、TypeScriptまたは最新のJavaScriptで書く。

## 除外ルール（やってほしくないこと）
- `gstack/` ディレクトリ配下にあるファイル（`setup`、`gen-skill-docs.ts` 等）の直接的な変更は絶対に行わないでください。`gstack/` はあくまでインプット（読み取り専用の素材）としてのみ扱ってください。
